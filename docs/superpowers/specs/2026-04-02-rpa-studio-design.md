# RPA Studio — 설계 문서

## 개요

Windows 데스크톱 앱과 브라우저 웹앱을 대상으로 하는 유저 친화적 RPA(Robotic Process Automation) 도구. 비개발자 직장인이 코딩 없이 반복 업무를 자동화할 수 있도록 설계한다.

### 대표 사용 시나리오

**Microsoft Teams Task 백업 자동화 (매월 1일 반복):**
1. Teams 열기
2. 완료된 Task 목록 선택
3. 엑셀로 백업 복사
4. 매크로 파일로 저장
5. 완료된 업무 일괄 삭제
6. 알림 표시

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 언어 | Python 3.11+ | 자동화 라이브러리 생태계, 빠른 프로토타이핑 |
| GUI | PyQt6 | 네이티브 데스크톱, 크로스 플랫폼 가능성 |
| UI Automation | pywinauto | Windows UI Automation API 래퍼 (1차 식별), comtypes UIAutomationClient 직접 호출은 필요 시 보조 |
| 키보드/마우스 | pyautogui, pynput | pyautogui: 좌표 기반 입력, pynput: 글로벌 핫키 및 녹화 이벤트 감지 |
| 엑셀 | openpyxl | .xlsx 읽기/쓰기 |
| 스케줄링 | APScheduler | 크론식 스케줄, 백그라운드 실행 |
| 파일 감시 | watchdog | 파일/폴더 변경 감지 트리거 |
| 시스템 트레이 | pystray | 트레이 아이콘 상주 |
| 빌드 | PyInstaller | .exe 패키징 |
| 이미지 매칭 | opencv-python-headless | (2차 구현 예정) |
| OCR | (미정) | (2차 구현 예정) |

> **플랫폼 제한**: 이 도구는 **Windows 전용**입니다. 자동화 레이어(pywinauto, UI Automation API)가 Windows에 종속됩니다.

- 실행 후 대기 시간 (초)
- 도움말 박스

### 요소 선택기 (UI Automation)

"🎯 요소 선택하기" 클릭 시:
1. RPA Studio 창이 최소화됨
2. 마우스를 대상 앱 위에 올리면 해당 UI 요소에 하이라이트 (빨간 테두리)
3. 클릭하면 해당 요소 정보 캡처:
   - Name, ClassName, AutomationId, ControlType
   - 부모→자식 경로 (요소 식별용)
4. RPA Studio 복원, 속성 패널에 선택 결과 표시
5. ESC 키로 선택 취소 가능

**요소 식별 전략 (resilience)**:
- 1순위: AutomationId (가장 안정적, 앱 업데이트에도 유지되는 경우 많음)
- 2순위: Name + ControlType 조합
- 3순위: 부모→자식 경로 (XPath 스타일)
- 실행 시 1순위부터 시도, 실패하면 다음 전략으로 폴백
- 요소를 찾지 못하면: "2번째 단계의 '완료됨 필터' 버튼을 찾을 수 없어요. 앱 화면이 바뀌었을 수 있습니다. 요소를 다시 선택해주세요."

### 레코더

"⏺ 녹화" 클릭 시:
1. RPA Studio 최소화
2. 화면 상단에 **플로팅 녹화 툴바** 표시 (항상 최상위, 반투명):
   - [⏹ 녹화 중지 (F9)] [⏸ 일시정지] [녹화 시간: 00:12]
3. 사용자의 마우스 클릭, 키보드 입력, 앱 전환 등을 `pynput` 리스너로 감지
4. 각 동작을 실시간으로 스텝 리스트에 추가
5. **F9 글로벌 핫키** 또는 플로팅 툴바 클릭으로 녹화 종료
6. RPA Studio 복원, 생성된 스텝 리스트를 편집 가능

**글로벌 핫키**: F9 (녹화 시작/중지 토글). F9이 대상 앱과 충돌할 경우 설정에서 변경 가능.

녹화 감지 항목:
- 마우스 클릭 → "UI 요소 클릭" 스텝 (pywinauto로 클릭 위치의 UI 요소 식별)
- 키보드 입력 → "텍스트 입력" 또는 "단축키" 스텝
- 앱 전환 → "창 전환" 스텝
- 대기 시간 → "기다리기" 스텝 (3초 이상 대기 시 자동 삽입)

## 실행 엔진

### 스텝 실행기

```python
class StepExecutor:
    def __init__(self):
        self._stop_event = threading.Event()

    def run(self, project: Project, context: ExecutionContext):
        for step in project.steps:
            if self._stop_event.is_set():
                break
            context.current_step = step
            self.on_step_enter(step)
            result = step.action.execute(context)
            self.on_step_exit(step, result)
            if step.wait_after > 0:
                # 인터럽트 가능한 대기 (중지 버튼 즉시 반응)
                self._stop_event.wait(timeout=step.wait_after)

    def stop(self):
        self._stop_event.set()
```

- 스텝을 순차적으로 실행
- 반복/조건 스텝은 자식 스텝을 재귀 실행
- `ExecutionThread`(QThread)로 GUI와 분리, Qt 시그널로 GUI 업데이트
- `threading.Event`로 인터럽트 가능한 대기 — 중지 버튼 즉시 반응
- 콜백: `on_step_enter`, `on_step_exit`, `on_log`, `on_error`
- 최대 실행 스텝 수 제한: 100,000 (무한루프 방지)
- 최대 중첩 깊이: 10단계

### 실행 컨텍스트

```python
class ExecutionContext:
    variables: dict[str, str | int | float | list]  # 저장값 (사용자 변수)
    running: bool            # 실행 중 플래그
    current_step: Step       # 현재 실행 중인 스텝
    log: list[str]           # 실행 로그
    error: Optional[str]     # 에러 메시지
```

**저장값(변수) 타입**: 문자열, 숫자(정수/소수), 목록. 타입은 자동 추론하며 사용자에게 노출하지 않음.
**저장값 참조 문법**: 액션 파라미터에서 `{이름}`으로 참조. 예: 파일 경로에 `{백업폴더}/결과.xlsx`.
속성 패널에서는 드롭다운으로 기존 저장값을 선택하거나 직접 입력 가능.

### 동시 실행 제어

- 같은 프로젝트는 동시에 1개만 실행 가능 (프로젝트별 잠금)
- 스케줄러가 실행을 시도할 때 이미 실행 중이면 건너뛰고 로그에 기록
- 스케줄러/트리거 실행은 GUI 없이도 동작 (ExecutionContext가 독립적)
- 스케줄러 스레드 → Qt 메인 스레드 통신은 `QMetaObject.invokeMethod` 또는 커스텀 시그널 사용

### 에러 처리

- 각 액션 실행 시 try/except로 에러 캡처
- 에러 발생 시: 로그에 자연어 에러 메시지 기록, 실행 중단 또는 다음 스텝 계속 (설정 가능)
- 에러 메시지 예: "3번째 단계에서 문제가 생겼어요: Teams 앱을 찾을 수 없습니다. 앱이 실행 중인지 확인해주세요."
- 실행 로그는 디스크에도 저장: `~/.rpa_studio/logs/YYYY-MM-DD_HH-MM-SS_프로젝트명.log`
- 스케줄/트리거 무인 실행 시 로그 파일이 유일한 디버깅 수단

## 스케줄러 & 트리거

### 크론 스케줄러

- APScheduler의 `BackgroundScheduler` 사용
- 시스템 트레이에 상주하며 앱 메인 윈도우가 닫혀도 동작
- 스케줄 정보는 프로젝트 파일에 JSON으로 저장

### 트리거

| 트리거 유형 | 감지 방법 | 예시 |
|------------|----------|------|
| 앱 실행 감지 | 프로세스 목록 폴링 (5초 간격) | "Teams.exe 실행되면 매크로 시작" |
| 파일 생성 감지 | watchdog 라이브러리 | "다운로드 폴더에 .csv 파일 생기면 실행" |

### 시스템 트레이

- pystray로 트레이 아이콘 상주
- 메뉴: 열기, 스케줄 관리, 실행 중 작업, 종료
- 스케줄/트리거 실행 시 트레이 알림
- **창 닫기 = 트레이로 최소화** (프로세스 유지, 스케줄/트리거 계속 감시)
- "종료" 메뉴를 통해서만 완전 종료

### 앱 프로세스 매칭

"앱 열기", "앱 감지 트리거" 등에서 앱을 식별하는 방법:
- 사용자에게는 앱 이름(예: "Microsoft Teams")을 입력받음
- 내부적으로 `psutil`로 프로세스 이름 매칭 (Teams.exe, ms-teams.exe 등)
- 시작 메뉴/바탕화면 바로가기에서 실행 경로를 자동 탐색
- 앱 이름 → 프로세스 매핑은 내장 사전 + 사용자 커스텀 설정
